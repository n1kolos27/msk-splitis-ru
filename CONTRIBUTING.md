# Руководство по разработке

Спасибо за интерес к проекту msk.splitis.ru!

## Структура проекта

Проект использует **Eleventy (11ty) v3.1.2** как статический генератор сайтов.

```
├── src/
│   ├── pages/          # HTML страницы с front matter (YAML)
│   ├── _includes/      # Шаблоны (layouts)
│   └── _data/          # Централизованные данные (JSON)
│
├── assets/             # Ресурсы (CSS, JS, изображения)
├── components/         # HTML компоненты
├── api/               # API endpoints для форм
├── tests/             # E2E тесты (Playwright)
└── _site/             # Собранный сайт (генерируется)
```

## Правила разработки

### HTML и Front Matter

- Используйте семантические HTML5 теги
- Все страницы должны иметь front matter с `layout: page.html`
- Добавляйте уникальные `title` и `description` для SEO
- Используйте правильную иерархию h1-h6 (один h1 на страницу)
- Добавляйте `alt` теги для всех изображений

Пример front matter:
```yaml
---
layout: page.html
permalink: /my-page.html
title: "Заголовок страницы"
description: "Описание страницы для SEO"
---
```

### CSS

- Следуйте **Mobile-first** подходу
- Используйте CSS переменные из дизайн-системы
- Компонентные стили в `assets/css/components/`
- Утилитарные классы в `assets/css/utilities/`
- Поддерживайте читаемость кода
- Комментируйте сложные секции

### JavaScript

- Используйте ванильный JavaScript (без зависимостей)
- Модульная структура в `assets/js/`
- Все скрипты должны иметь `defer` атрибут
- Проверяйте существование элементов перед использованием

### Данные

- Контактная информация: `src/_data/contacts.json`
- Настройки сайта: `src/_data/site.json`
- Навигация: `src/_data/navigation.json`

### Именование

- Используйте **kebab-case** для файлов и классов
- Используйте **BEM-подобную** методологию для CSS классов
- Делайте имена понятными и описательными

## Процесс разработки

1. **Клонируйте репозиторий**
   ```bash
   git clone <repository-url>
   cd msk-splitis-ru
   ```

2. **Установите зависимости**
   ```bash
   npm install
   ```

3. **Создайте ветку для вашей функции**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Запустите dev сервер**
   ```bash
   npm run dev
   ```

5. **Внесите изменения**
   - Редактируйте файлы в `src/pages/`
   - Проверьте результат в браузере (обычно http://localhost:8080)

6. **Проверьте валидность**
   - HTML: W3C Validator
   - CSS: W3C CSS Validator
   - Запустите тесты: `npm test`

7. **Соберите проект**
   ```bash
   npm run build
   ```

8. **Создайте Pull Request**
   - Опишите изменения
   - Укажите связанные issues (если есть)

## Тестирование

Проект включает E2E тесты на Playwright:

```bash
# Запустить все тесты
npm test

# Запустить тесты с UI
npm run test:ui

# Запустить тесты в debug режиме
npm run test:debug
```

## SEO требования

- Уникальный `title` (50-60 символов) для каждой страницы
- Уникальный `description` (150-160 символов)
- Schema.org разметка (JSON-LD) в front matter
- Правильная структура заголовков (h1 → h2 → h3)
- Alt теги для всех изображений
