# Руководство по разработке

Спасибо за интерес к проекту!

## Структура проекта

Проект организован следующим образом:

```
public_html/
├── assets/          # Ресурсы (CSS, JS, изображения)
├── components/      # HTML компоненты для включения
├── katalog/        # Страницы каталога
├── brands/         # Страницы брендов
├── uslugi/         # Страницы услуг
├── blog/           # Статьи блога
└── docs/           # Документация
```

## Правила разработки

### HTML
- Используйте семантические HTML5 теги
- Поддерживайте валидность HTML5
- Добавляйте alt теги для всех изображений
- Используйте правильную иерархию h1-h6

### CSS
- Следуйте Mobile-first подходу
- Используйте CSS переменные для цветов
- Поддерживайте читаемость кода
- Комментируйте сложные секции

### Именование
- Используйте kebab-case для файлов и классов
- Используйте BEM-подобную методологию для CSS классов
- Делайте имена понятными и описательными

## Процесс разработки

1. Создайте ветку для вашей функции
2. Внесите изменения
3. Проверьте валидность HTML/CSS
4. Протестируйте на разных устройствах
5. Создайте Pull Request
